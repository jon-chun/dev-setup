### Fixing Nerd Fonts in iTerm2 and VSCode Terminal on macOS

Since your JetBrains Mono Nerd Font works in Ghostty but shows junk characters (likely placeholder boxes or garbled icons) in iTerm2 and VSCode, the issue is probably that those apps aren't configured to use the Nerd Font variant. The font itself is likely installed correctly system-wide (via Homebrew or manual download), but each terminal emulator needs explicit font selection for proper glyph rendering, especially for Oh My Zsh themes or plugins like Powerlevel10k that rely on Nerd Font icons.

Here's how to fix it step by step for each app. After making changes, quit and restart the app (or reload the terminal session with `source ~/.zshrc` if needed).

#### 1. Verify the Font Installation and Name
   - Open the **Font Book** app on macOS (search in Spotlight).
   - Search for "JetBrains" or "Nerd".
   - Look for entries like "JetBrainsMono Nerd Font" (or variants like "JetBrainsMonoNerdFont-Regular"). Note the exact family name—this is what you'll use in settings. If nothing shows up, reinstall the font:
     - Via Homebrew: `brew tap homebrew/cask-fonts && brew install --cask font-jetbrains-mono-nerd-font`
     - Or download from the Nerd Fonts site (nerdfonts.com) and double-click the .ttf/.otf files to install.

#### 2. Configure iTerm2
   - Open iTerm2.
   - Go to **iTerm2 > Settings** (or press Cmd + ,).
   - Click the **Profiles** tab on the left.
   - Select your active profile (usually "Default").
   - Switch to the **Text** sub-tab.
   - In the **Font** section, click the font selection button (it might currently show a default like "Monaco").
   - Search for "JetBrainsMono Nerd Font" (or the exact name from Font Book).
   - Select a style like "Regular" or "Medium" (monospace is key for terminals).
   - Optional: Check **Use ligatures** if your theme benefits from it, but disable **Anti-aliased** if rendering looks off.
   - Close settings and test by opening a new tab/session. Icons should now render correctly.

   If some specific icons still fail, it could be a font variant issue—try a different Nerd Font like "Hack Nerd Font" as a fallback.

#### 3. Configure VSCode Integrated Terminal
   - Open VSCode.
   - Go to **Code > Settings** (or press Cmd + ,).
   - In the search bar, type "terminal.integrated.fontFamily".
   - In the **Terminal > Integrated: Font Family** field, enter `'JetBrainsMono Nerd Font'` (use single quotes if needed, and match the exact family name from Font Book).
     - Example: If Font Book shows "JetBrainsMonoNerdFont-Regular", use `'JetBrainsMonoNerdFont-Regular'`.
   - Optional: Set **Terminal > Integrated: Font Size** to 12–14 for better icon visibility.
   - Save and open a new integrated terminal (View > Terminal or Ctrl + `).
   - If icons persist as junk, add a fallback: `'JetBrainsMono Nerd Font', monospace'`.

   Note: This only affects the integrated terminal, not the editor itself. If you want Nerd Fonts in the editor too, search for "editor.fontFamily" and set it similarly.

#### Additional Tips
- **Restart Everything**: After changes, fully quit iTerm2 and VSCode (right-click dock icon > Quit), then reopen. Sometimes, macOS font caching lags—run `atsutil databases -removeUser && atsutil server -shutdown && atsutil server -ping` in Terminal to refresh the system font cache.
- **Test with a Command**: Run `echo -e "\ue0b0 \u00b1 \ue0a0 \u27a6 \u2699"` in each terminal. This should display common Nerd Font icons (like arrows and gears) if fixed.
- **Common Pitfalls**: Ensure your Oh My Zsh config (e.g., ~/.zshrc or Powerlevel10k theme) is set to use Nerd Font-compatible prompts. If using Powerlevel10k, rerun `p10k configure` and select a Nerd Font option.
- If issues continue, try an alternative Nerd Font like Fira Code or Meslo for testing—install via Homebrew and swap in the settings above.
